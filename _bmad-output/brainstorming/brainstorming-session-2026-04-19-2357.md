#### 🏪 Domain: Storefront UX & Design

**[Storefront #1]**: Age-Filtered Browsing
_Concept_: Parent enters child's age on arrival. Storefront auto-shows only size-appropriate clothing. No irrelevant products.
_Novelty_: Most stores make parents hunt for sizes — this flips the experience.

**[Storefront #2]**: Video-First Product Cards
_Concept_: Each product card shows a 3-5 sec autoplay clip of a child wearing the item in movement, not posed. Thumbnail photos on hover-off.
_Novelty_: Clothes look very different on a moving kid vs. a static photo — this closes that gap.

**[Storefront #3]**: Mix & Match Preview Mode
_Concept_: Tap multiple items and see them styled together as an outfit before messaging seller.
_Novelty_: Parents often buy outfits, not individual pieces — make that the primary action.

**[Storefront #4]** ⭐ SELECTED: WhatsApp Message Pre-Filled
_Concept_: Clicking a product opens WhatsApp with a pre-written message: "Hi! I'm interested in [Product Name] – Size [X] in [Color]. Is it available?"
_Novelty_: Removes friction entirely — parent doesn't have to type anything to start.

**[Storefront #5]** ⭐ SELECTED: Telegram Product Bot
_Concept_: Clicking redirects to your Telegram bot, not just your account. Bot confirms availability, shows more images, and collects order info automatically.
_Novelty_: Scales you — one person can handle 10x orders because the bot does intake.

**[Storefront #6]**: WhatsApp Catalog Integration
_Concept_: Storefront syncs with your WhatsApp Business Catalog, so the same products show in both places. One source of truth.
_Novelty_: No double-updating inventory across two platforms.

**[Storefront #7]** ⭐ SELECTED: "Saved for Later" via WhatsApp
_Concept_: A "Save" button sends the product link to your own WhatsApp as a saved message — so parents can share it with their partner or revisit later.
_Novelty_: Uses WhatsApp's own "saved messages" as a free wishlist.

**[Storefront #8]**: Seasonal Collections Landing Page
_Concept_: Homepage shows "Back to School," "Easter," "Summer Play" — click into a curated seasonal collection.
_Novelty_: Parents think in seasons & occasions, not product categories.

**[Storefront #9]** ⭐ SELECTED: Size Guide Built into Every Product
_Concept_: Each product card has a collapsible "What size is my child?" quick guide (age → typical size mapping).
_Novelty_: The #1 friction point in kids' clothing is size uncertainty — solve it at point of interest.

**[Storefront #10]**: Nearly Gone Urgency Tags
_Concept_: If a product has ≤3 left in stock, it auto-shows a "⚡ Only 2 left!" badge.
_Novelty_: Creates real urgency without fake countdown timers — builds trust.

**[Storefront #11]**: "Ask Before You Buy" Button
_Concept_: Every product has both "Buy via WhatsApp" AND "Ask a Question via WhatsApp" — different pre-filled messages.
_Novelty_: Lowers commitment bar — even hesitant parents engage.

**[Storefront #12]**: Bundle Deals Displayed as One Card
_Concept_: "3-piece play set – $25" shown as a single card. Click → WhatsApp shows all 3 items in the message.
_Novelty_: Increases average order value naturally through bundle framing.

#### 💬 Domain: WhatsApp/Telegram Handoff

**[Handoff #21]** ⭐ SELECTED: WhatsApp Opening Line = Relationship Start
_Concept_: The pre-filled WhatsApp message ends with a personal touch that makes your reply feel like a continuation of a real conversation, not a transaction.
_Novelty_: The first message sets the emotional tone of the entire transaction.

_Concept_: Every reply includes one extra value line — fit advice, care tip, pairing suggestion — making you their trusted advisor instantly.
_Novelty_: This single habit builds repeat customers better than any discount.

**[Handoff #23]**: Response Time Badge on Storefront
_Concept_: Display "⚡ Typically replies within 1 hour" on storefront. Parents know what to expect.
_Novelty_: Reduces abandonment from uncertainty — silent conversion killer addressed.

**[Handoff #24]** ⭐ SELECTED: "Currently Available" Online Status
_Concept_: A small green dot or "I'm online now — reach out!" banner when actively available to respond. Like live chat, but human.
_Novelty_: Real-time availability signals transform browsing to buying.

**[Handoff #25]**: Personal Thank You After Purchase
_Concept_: After every order, send a 15-second WhatsApp voice note saying thank you by name and mentioning the child's name.
_Novelty_: Voice notes feel like calls from a friend. Zero other sellers do this.

**[Handoff #26]** ⭐ SELECTED: "Remember Me" Follow-Up System
_Concept_: After an order, note child's age/size. In 3 months proactively message: "Hey! Your little one is probably ready for the next size up — we just got gorgeous new pieces."
_Novelty_: Pro-active re-engagement tied to child growth = automatic repeat business.

**[Handoff #27]**: Exclusive "First Look" for Past Buyers
_Concept_: Every time new stock arrives, previous customers get a WhatsApp message first — before the storefront is updated. "You get first pick!"
_Novelty_: Makes past buyers feel like VIPs. Creates a loyalty loop at zero cost.

**[Handoff #28]**: Offer 2 Platforms — Customer Chooses
_Concept_: Every product has TWO buttons: "📱 Chat on WhatsApp" and "✈️ Message on Telegram" — customer picks their comfort zone.
_Novelty_: Removes platform friction — some parents live in Telegram, some in WhatsApp.

#### Table 2: `products`
```sql
CREATE TABLE products (
  id             INT AUTO_INCREMENT PRIMARY KEY,
  name           VARCHAR(255)  NOT NULL,
  description    TEXT          NULL,             -- short paragraph, optional
  price          INT           NOT NULL,         -- current price in ₦ (no decimals)
  original_price INT           NULL,             -- pre-discount price; NULL = no sale
  category_id    INT           NOT NULL,         -- FK → categories.id
  gender         ENUM('girls','boys','unisex') NOT NULL DEFAULT 'unisex',
  material       VARCHAR(255)  NULL,             -- display if provided
  brand          VARCHAR(255)  NULL,             -- display if provided
  tiktok_url     VARCHAR(500)  NULL,             -- optional TikTok video link
  season_tag     VARCHAR(100)  NULL,             -- optional e.g. "Christmas", "Eid"
  available      TINYINT(1)    NOT NULL DEFAULT 1, -- 0 = hidden entirely
  created_at     TIMESTAMP     DEFAULT CURRENT_TIMESTAMP,
  updated_at     TIMESTAMP     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (category_id) REFERENCES categories(id)
);
```

**Price logic:**
- `original_price IS NULL` → display: **₦5,500**
- `original_price IS NOT NULL` → display: ~~₦8,000~~ **₦5,500** + "SALE" badge

#### Table 3: `product_images`
```sql
CREATE TABLE product_images (
  id          INT AUTO_INCREMENT PRIMARY KEY,
  product_id  INT           NOT NULL,
  image_path  VARCHAR(500)  NOT NULL,   -- e.g. "/uploads/products/42/img1.jpg"
  sort_order  INT           DEFAULT 0,  -- 0 = primary/thumbnail
  FOREIGN KEY (product_id) REFERENCES products(id) ON DELETE CASCADE
);
```

**Notes:**
- Unlimited images per product
- `sort_order = 0` = primary thumbnail shown on product card
- Images stored in `/public_html/uploads/products/{product_id}/`

---

#### Table 4: `product_sizes`
```sql
CREATE TABLE product_sizes (
  id           INT AUTO_INCREMENT PRIMARY KEY,
  product_id   INT          NOT NULL,
  size_label   VARCHAR(20)  NOT NULL,  -- e.g. "3-4Y", "5", "6-12M", "XS"
  stock_count  INT          NOT NULL DEFAULT 0,
  sort_order   INT          DEFAULT 0, -- controls display order
  FOREIGN KEY (product_id) REFERENCES products(id) ON DELETE CASCADE
);
```

**A product is "Sold Out" when ALL its size rows have `stock_count = 0`.**
**A size is "Sold Out" when its own `stock_count = 0`.**
